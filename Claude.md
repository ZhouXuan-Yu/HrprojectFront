> 定义 agent 如何干活、代码风格是什么、哪些事情不要做。用户口头提到 `Cluade.md` 时，统一视为本文件 `Claude.md`。

## 1. 项目身份

你是「智能招聘系统」前端项目的协作 agent。当前目标不是重做一个新系统，而是在保留既有功能、页面和业务口径的前提下，把原型逐步工程化、Vue 化、可测试化，并保持企业级视觉质量。

当前项目位置：

- 工作目录：`D:\WorkProject\HrProject\hr-web\frontend`
- 技术栈：Vue 3 + Vite + Vue Router + Playwright
- 现状：已有 Vue shell 包装 legacy 页面，业务页面仍在 `public/legacy/`
- 登录页：已改为原创「人才雷达中控台」动效登录页
- Git：本地仓库已建立，重要节点已提交

## 2. 工作方式

每次接手任务先做四件事：

1. 读 `Memory.md`，确认当前进度和下次继续点。
2. 查 `Wiki.md`，确认业务口径、角色权限和技术边界。
3. 若涉及风险、反复返工或用户纠正，更新 `Learning.md`。
4. 改完代码后，跑可用验证，并按“三轮修改规则”维护四个文档。

优先级：

1. 保功能不变。
2. 保数据流和路由不变。
3. 保企业级视觉和可访问性。
4. 再考虑动效、装饰和体验增强。

## 3. 代码风格

### Vue / JavaScript

- 优先沿用项目现有入口和路由结构，不擅自拆大架构。
- legacy 页面迁移时，先保持行为一致，再拆组件。
- 副作用集中处理，避免散落全局变量。
- 本地存储键名沿用现有约定：`hr_role`、`hr_user`。
- 不引入新依赖，除非用户明确同意或现有能力无法完成。

### HTML

- 使用语义化标签：`main`、`nav`、`aside`、`button`、`label`。
- 表单字段必须有可见 `label`。
- 可点击元素必须是 `button` 或 `a`，不要用 `div onclick`。
- 登录、弹窗、抽屉等关键路径要考虑键盘可达和 focus 状态。

### CSS

- 后台业务页优先使用既有 `css/style.css` token。
- 单页特殊视觉可以使用页面内 scoped 样式，但不要污染全局。
- 企业工具页避免过度装饰：少渐变、少大阴影、少营销式 hero。
- 圆角保持克制：卡片和表单通常 8px 左右，大圆角只用于输入框和明确控件。
- 动效只使用 `transform`、`opacity`、CSS 变量等低风险属性。
- 必须支持 `prefers-reduced-motion`。

### 测试

- 功能改动后跑 `npm test`。
- 构建或依赖相关改动后跑 `npm run build`。
- UI 改动至少截一张桌面图；影响移动端时也截移动图。
- 重要交互用 Playwright 覆盖，避免只靠目测。

## 4. 禁止事项

| 不要做 | 正确做法 |
|---|---|
| 未经确认删除、移动用户文件 | 只新增或精确修改；删除必须先确认 |
| 为视觉效果复制 GitHub demo | 只参考交互原理，视觉资产和代码重新设计 |
| 让页面有明显 AI 生成味 | 使用真实业务语境、克制文案和企业级布局 |
| 把 AI 写成自动打电话/外呼 | AI 只做联系话术、沟通摘要和信息补齐辅助 |
| 重写登录逻辑、权限、路由 | 保持 localStorage 和跳转行为不变 |
| 引入 Tailwind/CDN/动画库 | 先用现有 HTML/CSS/JS 完成 |
| 把未请求的 Markdown、缓存、报告混入提交 | 只提交本次任务相关文件 |
| 用 emoji 冒充图标 | 用文本、CSS 或正式图标方案 |
| 编造真实业务数据 | 样例必须是原型数据或明确标注 |

## 5. 文档维护规则

四个长期文档：

- `Claude.md`：agent 工作说明和代码风格。
- `Memory.md`：当前进度、刚确认的事情、下次从哪里继续。
- `Learning.md`：复盘记录，记录哪里错了、为什么错、下次如何改。
- `Wiki.md`：长期稳定的项目常识、业务口径、接口和架构说明。

三轮修改规则：

- “一轮修改”指一次有实际产出的代码、设计或文档改动。
- 每累计 3 轮修改，必须更新 `Memory.md`。
- 若三轮内出现返工、用户纠正、测试失败或判断失误，必须同步更新 `Learning.md`。
- 若三轮内确认了长期稳定的业务口径、架构或接口，必须同步更新 `Wiki.md`。
- 若工作方式或代码规范发生变化，必须同步更新 `Claude.md`。
