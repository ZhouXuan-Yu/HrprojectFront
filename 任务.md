# UI 优化改造任务规划

> 目标：按照 `DESIGN.md` 的“千亿级管理后台”标准，分阶段优化当前 HR 前端项目。每一阶段结束后必须完成构建、自动化测试、模拟操作、截图巡检和对抗性审查。

## Phase 0：现状冻结与审计

### 目标

- 冻结当前功能基线、路由、页面、测试截图。
- 梳理所有页面的主任务、次任务、危险操作、空状态、加载状态、错误状态。
- 输出当前 UI 问题清单：信息层级、表格密度、交互状态、视觉一致性、AI 味、企业感不足等。

### 验收

- `npm run build` 通过。
- `npm test` 通过。
- 主要页面截图已生成。
- 路由、权限、登录、筛选、弹窗、抽屉、面试、配置等功能可正常模拟。

### 当前状态

- 已完成。
- 当前基线：Playwright 21/21 通过。

## Phase 1：设计 Token 层

### 目标

- 建立统一颜色、字号、间距、圆角、阴影、边框、状态色。
- 采用“企业级浅色工作台 + 可扩展暗色指挥中心”的方案。
- 所有页面禁止继续散落硬编码颜色，统一通过 CSS variables 管理。
- 建立表格、表单、按钮、标签、状态、KPI 数字的基础视觉规范。

### 开发任务

- 统一维护 `public/css/style.css` 中的 `--e-*` 和 `--c-*` token。
- 现有页面优先通过 token 覆盖层升级，不破坏 legacy 页面结构。
- Vue 壳层 loading/error/focus 等状态必须复用全局 token。
- 新增颜色、阴影、状态样式时必须先定义 token，再使用 token。
- 遗留内联硬编码颜色作为迁移债务记录，后续随组件化逐步消除。

### 验收

- `npm run build` 通过。
- `npm test` 通过。
- 检查新增代码不引入新的散落硬编码颜色。
- 桌面/移动截图无明显错位。

### 当前状态

- 已完成。
- 本阶段完成内容：补齐 `任务.md`，并将 Vue legacy loading/error 状态 token 化。
- 验证结果：`npm run build` 通过，`npm test` 通过，21/21。

## Phase 2：全局框架重构

### 目标

- 优化侧边栏、顶部栏、页面标题区、面包屑、用户区、全局搜索/命令入口。
- 页面从“卡片堆叠感”改成“工作台布局”：左导航、上筛选、中央数据、右侧详情/审计。
- 减少装饰性元素，强化可扫描、可比较、可操作。

### 开发任务

- 复审并深化全局外壳：侧栏、顶栏、内容区、命令入口、移动端导航必须保持一致。
- 为页面外壳补齐可审计结构：当前路由标记、主内容区域、导航语义、当前页状态。
- 顶栏动作统一归入 `topbar-actions`，避免各页面控件散落和窄屏撑裂。
- `Ctrl/Cmd + K` 命令入口需要同时支持键盘和按钮点击，并按角色权限展示可访问页面。
- 移动端保持管理后台形态：导航横向扫描、顶栏动作可滚动、页面级无横向溢出。

### 验收

- 侧边栏与顶部栏在所有主页面一致。
- `Ctrl/Cmd + K` 快速跳转可用。
- 桌面和移动端布局不横向撑开页面。
- `npm run build` 通过。
- `npm test` 通过。
- 截图巡检 console error 为 0，移动端横向溢出为 0。

### 当前状态

- 已完成复审与深化。

## Phase 4：页面逐个改造

### 目标

- 登录页：保留项目专属动态识别感，但压低炫技感，做成专业 HR 系统入口。
- 招聘看板：改成经营驾驶舱，突出招聘漏斗、待处理事项、异常预警、最近动作。
- 需求管理：改成标准企业列表页，强化筛选、批量处理、状态流转、负责人协作。
- 需求详情：改成对象详情页，顶部对象信息，中部 tabs，右侧流程/审计/备注。
- 人才库：改成高密度人才资产管理页，支持视图筛选、标签、进度、最近联系记录。
- 面试管理：改成任务流页面，区分待安排、待面试、待评价、待录用、已结束。
- AI 页面：去掉不现实的“AI 外呼”表达，替换为候选人沟通辅助、简历摘要、话术建议、招聘效率分析。
- 配置页面：改成左侧配置导航 + 中央表单 + 右侧影响说明/审计记录。

### 验收

- 每个页面主任务清楚。
- 不出现营销页味道、AI 味废话或不可验证能力承诺。
- 保留现有功能和测试入口。

## Phase 5：交互与可访问性打磨

### 目标

- 补齐 hover、active、disabled、focus-visible、loading 状态。
- 所有关键操作加确认、撤销或风险提示。
- 支持键盘导航，保证表单、按钮、抽屉、弹窗可操作。
- 移动端不强行塞满，列表转为重点信息卡片或横向表格。

### 验收

- 键盘可完成主要路径。
- 移动端无页面级横向溢出。
- 支持 `prefers-reduced-motion`。

## Phase 6：质量门禁与对抗性审查

### 目标

- 跑现有自动化测试。
- 用 Playwright 模拟完整操作流：登录、看板、需求、详情、人才库、面试、AI、配置。
- 检查控制台错误、路由异常、按钮不可点、弹窗无法关闭、筛选失效、表格错位。
- 按 `DESIGN.md` 的 100 分标准评分，目标不低于 96，低于 92 不交付。
- 明确检查：不能有营销页味道、不能有 AI 味废话、不能有无意义渐变装饰、不能出现“AI 外呼”。

### 验收

- `npm run build` 通过。
- `npm test` 通过。
- Console error 为 0。
- 移动端横向溢出为 0。
