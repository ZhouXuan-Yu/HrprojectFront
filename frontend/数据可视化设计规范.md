# 招聘管理系统 · 数据卡片与图表设计规范

> 用途：这是一份**可直接调用**的设计生成规范。需要新增 KPI 卡片、统计卡、趋势图、漏斗图时，
> 按本规范的 token、结构与代码片段直接生成即可，风格与企业级后台（白底、克制、高级）保持一致。
>
> 关键词：浅紫灰圆角方底 + 蓝色线性图标 / 白底统计卡 / 渐变面积图 / 玻璃圆锥漏斗。

---

## 1. 设计 Token（统一变量）

已定义于 `public/css/style.css`，生成时直接引用变量，不要写死色值。

| 变量 | 值 | 用途 |
|---|---|---|
| `--e-primary` | `#4F6EF7` | 主蓝：图标、数字强调、描边 |
| `--e-primary-subtle` | `rgba(79,110,247,.1)` | 浅紫灰图标底 |
| `--e-ink` | `#172033` | 大数字 / 标题 |
| `--e-ink-2` | `#344054` | 次级文字 |
| `--e-muted` | `#5B6475` | 正文辅助 |
| `--e-subtle` | `#8C97AA` | 副标题 / 说明 |
| `--e-border` | `#E1E6EF` | 卡片描边 |
| `--e-surface` | `#FFFFFF` | 卡片白底 |
| 圆角 | 卡片 `8px`，图标块 `8–10px` | 自然、不夸张 |
| 间距 | 卡片内边距 `18–20px`，卡间 `14–16px` | 统一节奏 |

字体：大数字用 `font-variant-numeric: tabular-nums`（等宽数字，对齐不跳动）。

## 2. KPI 图标（浅紫灰圆角方底 + 蓝色线性图标）

参考标准：**小巧精致、线宽适中（2）、圆角自然、配色柔和克制、不 3D、不拟物、白底适配**。

### 2.1 图标容器样式

```css
.metric-icon {
  width: 38px; height: 38px;
  border-radius: 8px;
  background: var(--e-primary-subtle); /* 浅紫灰 */
  color: var(--e-primary);             /* 蓝 */
  display: flex; align-items: center; justify-content: center;
  flex-shrink: 0;
}
```

### 2.2 图标 SVG 规格

- `viewBox="0 0 24 24"`，`width/height: 20px`
- `stroke: currentColor`（继承蓝色），`fill: none`
- `stroke-width: 2`，`stroke-linecap/linejoin: round`

```html
<svg viewBox="0 0 24 24" style="width:20px;height:20px;stroke:currentColor;fill:none;stroke-width:2;stroke-linecap:round;stroke-linejoin:round">
  <circle cx="12" cy="12" r="10"/><polyline points="12 6 12 12 16 14"/>
</svg>
```

### 2.3 直接复用：图标模块 `src/components/kpiIcons.js`

已实现，直接 import 调用，**按 label 自动配图标，数据无 icon 字段也能补齐**：

```js
import { resolveKpiIcon } from '../components/kpiIcons.js';
// 模板：<div class="metric-icon" v-html="resolveKpiIcon(kpi)"></div>
```

内置图标：`clock(默认) / calendar / edit / briefcase / userCheck / users / star / trendingUp / check / checkSquare / fileText`。
label 映射规则：面试→calendar、评价→edit、岗位→briefcase、入职→userCheck、均分→star、通过率→trendingUp、候选人→users、完成→check、兜底→clock。

> 后端 KPI 接口只返回 `val/label/trend`（无 icon），所以**必须**用 `resolveKpiIcon` 而不是 `kpi.icon`，否则图标盒为空。

## 3. 统计卡（推荐范式 · 人才库同款）

人才库「外部候选人 / 内部人才 / 黑名单 / 待联系」那组卡片是标准范式，**新统计卡一律仿照它**。
结构 = `span(标签) + strong(大数字) + em(副标题) + i(右上角图标块)`，底部一条 1px 分隔线。

### 3.1 HTML 结构

```html
<section class="hero-page-summary">
  <article class="hero-summary-card">
    <span>外部候选人</span>
    <strong>7</strong>
    <em>可筛选入库</em>
    <i></i>
  </article>
  <!-- …更多卡片 -->
</section>
```


> 图标块 `i` 内可放第 2 节的蓝色线性图标（推荐），替代默认的彩色圆点装饰，保持整套蓝色系一致。
> 卡片数不为 4 时，把 `grid-template-columns` 改为 `repeat(N, minmax(0,1fr))`。

## 4. 趋势面积图（HeroUI 风格 · 渐变面积 + 平滑曲线）

> 注：HeroUI Pro 的 `AreaChart` 是 React+Recharts，本 Vue 项目用**原生 SVG 复刻**，效果一致、零依赖。
> 特征：平滑单调曲线、纵向渐变填充（阶段色 22%→2%）、仅横向网格线、最值标注。

### 4.1 SVG 模板

```html
<svg viewBox="0 0 180 52" class="insight-spark-lg">
  <defs>
    <linearGradient id="sparkGrad" x1="0" y1="0" x2="0" y2="1">
      <stop offset="0%"  :stop-color="accent" stop-opacity="0.22"/>
      <stop offset="100%" :stop-color="accent" stop-opacity="0.02"/>
    </linearGradient>
  </defs>
  <line v-for="gy in [12,22,32]" :key="gy" x1="6" x2="174" :y1="gy" :y2="gy" class="spark-grid-line"/>
  <path :d="sparkAreaPath(data, 180, 40, 6)" fill="url(#sparkGrad)"/>
  <path :d="sparkLinePath(data, 180, 40, 6)" fill="none" :stroke="accent" stroke-width="2.5" stroke-linecap="round" stroke-linejoin="round"/>
</svg>
```

```css
.spark-grid-line{ stroke:rgba(79,110,247,.14); stroke-width:1; }
.insight-spark-lg{ width:100%; height:52px; display:block; }
```

### 4.2 平滑曲线工具函数（Catmull-Rom → Bezier）

见 `src/components/FunnelHero.vue` 中的 `sparkPts / smoothPath / sparkLinePath / sparkAreaPath`，可直接复制：

```js
function sparkPts(spark, w, h, pad){ /* 归一化数据点为 [x,y] 数组 */ }
function smoothPath(pts){ /* 相邻点用三次贝塞尔平滑连接，返回 path d */ }
function sparkLinePath(spark,w,h,pad){ return smoothPath(sparkPts(spark,w,h,pad)); }
function sparkAreaPath(spark,w,h,pad){ /* 平滑线 + 右下角 + 左下角闭合 */ }
```

要点：曲线用平滑贝塞尔而非折线；填充用渐变而非纯色；网格只画横向。

## 5. 3D 玻璃圆锥漏斗（Three.js）

`src/components/FunnelHero.vue` 已实现，要点（改动时勿破坏对齐）：

- **坐标对齐是生命线**：圆锥带（bands）与圆环（rings）必须用同一套 `DISC_Y` 坐标，不能一边 `-1.4` 一边不减——否则锥体与圆环错位，上部看似“没填充”。
- 结构：4 个开口圆柱 frustum 带（每段一个阶段色）连成圆锥 + 5 条细玻璃环（tube≈0.028）压在锥体轮廓作边界 + 粒子流在锥体**外侧**螺旋环绕。
- 材质：`MeshPhysicalMaterial`（transmission + clearcoat + 微 emissive），浅色底上半透明分层（Glassmorphism 2.0）。
- 配色：浅蓝→亮蓝→主蓝→蓝紫，入职描边用成功绿 `#22A06B`，**不要彩虹色、不要黑底**。
