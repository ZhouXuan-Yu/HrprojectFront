# 登录页面原创设计方案

## 目标

登录页需要有记忆点，但不能像通用 GitHub 动效 demo，也不能使用容易产生版权争议的动物形象、角色造型、文案或源码结构。本项目最终采用“人才雷达 / 招聘流程中控台”方向：把眼睛跟随鼠标的交互抽象成招聘系统里的双镜片扫描模块，背景表达招聘流程节点和候选人管线，整体仍然服务企业级 HR 工作台。

## 参考来源

- 参考仓库：`ZhouXuan-Yu/Eyes-Follow-Mouse-Login-Website`
- 地址：https://github.com/ZhouXuan-Yu/Eyes-Follow-Mouse-Login-Website.git

参考的不是具体视觉资产，而是交互原理：

- 鼠标移动时，页面记录光标相对位置。
- 眼睛或镜片根据坐标做小范围位移。
- 聚焦密码框时，视觉角色进入收敛或闭眼状态。
- 表单本身保持清晰，不让动效干扰登录任务。

## 原创化边界

为避免版权风险，本项目不复制参考仓库的以下内容：

- 不复制源码结构、类名体系、Tailwind 写法或布局代码。
- 不复制动物、多人角色、颜色组合、背景构图和角色比例。
- 不复制仓库中的中文文案、幽默表达或装饰性内容。
- 不引入原仓库图片、SVG、Logo 或外部资源。

本项目只保留通用交互思想：鼠标位置驱动 CSS 变量，再由 CSS 变量控制镜片方向、背景光晕和轻微视差。最终视觉资产全部使用项目内 HTML/CSS 重新绘制。

## 视觉方向

方向：企业级 Swiss 工具界面。

原因：这是 HR 招聘系统，不适合过度卡通化。白色表单、灰蓝品牌面板、清晰网格和克制动效更符合企业软件的信任感。

差异化动作：左侧“人才雷达中控台”不是动物，也不是机器人，而是一个招聘流程扫描模块。它的两个镜片跟随鼠标，聚焦账号时轻微靠近表单，聚焦密码时镜片收合，表达“隐私输入不被注视”。

关键视觉：

- 左侧灰蓝品牌面板，使用轻网格和招聘节点表达系统感。
- 中央原创“人才雷达中控台”，由双镜片模块、信息面板和管线节点组成。
- 右侧保持白底登录表单，角色选择、账号、密码、记住我和登录按钮不变。
- 主色沿用项目蓝 `#4F6EF7`，确保和后台页面一致。

## 交互方案

核心 CSS 变量：

- `--mx` / `--my`：鼠标在登录区域内的百分比位置。
- `--pupil-x` / `--pupil-y`：镜片内瞳点位移。
- `--tilt-x` / `--tilt-y`：左侧场景轻微 3D 倾斜。

状态：

- 默认：镜片看向鼠标，背景光晕跟随鼠标。
- `username-mode`：雷达中控台略向表单侧靠近，暗示正在读取账号。
- `password-mode`：镜片收合，降低密码输入时“被看见”的心理负担。
- 鼠标离开：所有变量回到中心位置。
- `prefers-reduced-motion: reduce`：关闭位移和视差过渡。

## 可访问性

- 保留可见 `label`，不使用 placeholder 替代标签。
- 增加跳转到登录表单的 skip link。
- 按钮、链接和输入框保留清晰 focus 样式。
- 移动端隐藏左侧动效面板，只展示登录表单。
- 动效遵守 `prefers-reduced-motion`。

## 实现约束

- 不新增外部依赖。
- 不引入图片、SVG 文件或第三方动画库。
- 不改变现有登录逻辑：角色选择、写入 `localStorage`、跳转 `recruit-dashboard.html`。
- E2E 继续覆盖鼠标响应、账号/密码聚焦状态、登录跳转和后台页面功能。

## 2026-07-17 UI v6 登录页打磨

本轮登录页不再追求“动态 demo 感”，而是贴合企业级 HR 系统入口：

- 保留原创双镜片跟随鼠标、账号/密码聚焦状态和动态背景。
- 左侧改为更克制的深色企业面板，减少装饰光效。
- 登录表单加入更明确的边框、层级和角色卡片选中态。
- 角色选择继续覆盖管理员、HR 专员、部门负责人、基层员工、面试官、临时面试官、无招聘权限。
- 验证通过：登录跳转、鼠标响应、账号/密码聚焦、角色菜单裁剪均由 Playwright 覆盖。
