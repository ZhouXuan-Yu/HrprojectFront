# 登录页动态设计方案

## 目标

当前登录页是静态白卡片，和招聘系统后续页面的企业工具气质不冲突，但缺少第一眼记忆点。改造目标是做成一个有鼠标响应的动态登录页：鼠标移动时背景的光晕、网格和节点轻微偏移，登录卡片保持清晰、稳定、可读。

## 使用的技能

- `frontend-design`：用于确定视觉锚点、内容纪律和整体页面气质。
- `css-animations`：用于设计可控的 CSS 动效，避免动画依赖不可预测的时间和布局变化。
- `design-trends`：用于提炼近期动态页面趋势，包括暗色主画布、鼠标互动、微动效、性能和可访问性约束。

## 设计方向

视觉方向选择企业级 split layout。原因：这是 HR 招聘工作台，登录页应当先让用户相信它稳定、清晰、可进入；互动角色只作为左侧品牌插画里的细节，而不是抢占表单注意力的主角。

## V3：企业级抽象角色登录页

参考仓库 `ZhouXuan-Yu/Eyes-Follow-Mouse-Login-Website` 的核心价值不是某个动物形象，而是“左侧品牌插画中的抽象角色会观察鼠标，右侧表单保持专业克制”。这一版去掉了过强的 AI 口吻和猫头鹰设定，改成更适合企业 HR 系统的 split layout。

参考实现：

- 左侧多角色场景，眼睛跟随鼠标移动。
- 密码输入时角色做出收敛/闭眼反应。
- 右侧表单非常干净，减少说明性文案。
- 背景以轻量网格、渐变和柔和光斑形成层次。

核心行为：

- 鼠标移动：页面记录 `--mx`、`--my`，左侧背景网格轻微偏移，角色瞳孔根据光标相对位置更新 `--pupil-x`、`--pupil-y`。
- 聚焦账号：登录页进入 `username-mode`，角色略微靠近表单方向。
- 聚焦密码：登录页进入 `password-mode`，角色眼睛收起，降低“被看见”的心理负担。
- 鼠标离开：所有 CSS 变量回到中位，角色恢复默认状态。
- 减少动画：`prefers-reduced-motion: reduce` 下关闭扫描线、节点漂浮、视差和瞳孔追踪。

技术实现：

- 使用纯 HTML/CSS 绘制抽象角色，不引入图片资源、Tailwind CDN 或外部动画库。
- 用 CSS transition 驱动角色位移、眼睛闭合和瞳孔移动。
- 用原生 `mousemove` 计算眼睛方向，限制瞳孔移动半径，避免视觉穿帮。
- 表单保持原逻辑：角色选择、账号写入 `localStorage`、跳转招聘看板。
- E2E 覆盖鼠标响应、账号/密码模式和登录跳转。

关键视觉：

- 左侧使用企业级灰蓝渐变和轻网格，右侧使用白色登录表单。
- 鼠标位置写入 CSS 变量 `--mx`、`--my`，驱动径向光晕、网格位移、节点偏移。
- 登录卡片取消厚重深色面板，回到专业白底表单。
- 主按钮使用项目原有品牌蓝 `#4F6EF7`，和后台页面视觉系统保持一致。
- 角色选择保留三种演示角色，改成三枚稳定的分段卡片。

## 交互策略

- 鼠标移动只改变 transform、background-position 和 CSS 变量，不动画布局属性。
- 背景有轻微常驻漂移，鼠标进入后产生更明显的视差。
- 输入框、角色卡、登录按钮提供 hover/focus 状态。
- `prefers-reduced-motion: reduce` 下关闭常驻动画和鼠标视差，保留静态层次。
- 移动端不依赖鼠标，背景使用中心光晕，表单垂直铺排。

## 实现约束

- 不新增外部库，避免登录页为了视觉效果引入额外运行时。
- 保留现有登录逻辑：选择角色、写入 `localStorage`、跳转招聘看板。
- 保留 E2E 可定位文案：`HR 专员`、登录按钮、账号和密码。
- 不改变业务页面的全局样式，只改 `public/legacy/login.html`。
