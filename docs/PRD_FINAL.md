## 一、菜单架构（统一层级，按角色裁剪）

### 1.1 固定结构

```
一级菜单：招聘管理（所有角色统一，悬浮/点击展开）
    ├── ① 招聘看板          /recruit/dashboard
    ├── ② 需求管理          /recruit/demand
    ├── ③ 人才库            /recruit/talent        （仅 HR / 管理员可见）
    ├── ④ 面试计划          /recruit/interview      （有面试权限时可见）
    ├── ⑤ AI 智能自动化中心  /recruit/ai-setting     （仅管理员可见，一期占位）
    └── ⑥ 招聘基础配置      /recruit/base-setting   （仅管理员可见，一期占位）
```

### 1.2 渲染规则

- 无权限菜单后端直接不返回，前端不渲染，不做置灰占位
- 所有角色菜单层级一致，代码复用同一套布局
- 面包屑格式：`招聘管理 > {当前页面名称}`

## 二、6 类角色完整权限

### 角色 1：普通基层业务员工

| 维度 | 内容 |
|------|------|
| 固定菜单 | 招聘看板、需求管理（2 项） |
| 看板 | 只看本人提交的需求进度，无部门全局数据 |
| 需求管理 | 只看自己提交的需求，无简历查看权限 |
| 隐藏 | 人才库、面试计划、AI、配置 |
| 被指定面试后 | 临时出现「面试计划」菜单，评价提交后自动回收 |

### 角色 2：业务部门负责人

| 维度 | 内容 |
|------|------|
| 固定菜单 | 招聘看板、需求管理（2 项） |
| 看板 | 本部门全部数据 + 本部门风险预警 |
| 需求管理 | 本部门所有需求、本部门需求审批权 |
| 查看简历 | 入口在需求详情页内的两个 Tab：①「外部候选人」Tab ②「内部可转岗员工」Tab |
| 操作限制 | 无发起面试、无邀约候选人、无独立人才库全局检索 |
| 被指定面试后 | 临时出现「面试计划」菜单 |
| 隐藏 | 人才库独立菜单、AI、配置 |

### 角色 3：临时面试官

| 维度 | 内容 |
|------|------|
| 权限来源 | HR 在需求管理中指定某人面试，系统自动授予 |
| 临时菜单 | 面试计划（仅此一项新增） |
| 数据范围 | 仅自己分配到的面试场次 + 该场候选人简历 |
| 操作 | 仅可填写面试评价、查看简历 |
| 权限回收 | 评价提交完成，即时收回菜单，恢复原身份 |
| 通知 | 系统自动短信 + 站内信提醒 |

### 角色 4：HR 招聘专员

| 维度 | 内容 |
|------|------|
| 固定菜单 | 招聘看板、需求管理、人才库、面试计划（4 项） |
| 数据范围 | 全公司无隔离——全部需求、全部候选人、全部面试 |
| 操作权限 | 新建需求、审批、渠道发布、筛选简历、发起面试、邀约、发 Offer |
| 隐藏 | AI、配置（2 项） |

### 角色 5：系统管理员

| 维度 | 内容 |
|------|------|
| 固定菜单 | 全部 6 个二级菜单 |
| 数据范围 | 全量无隔离 |
| 配置权限 | 渠道管理、审批流程、Offer 模板、AI 规则、简历封存规则、角色权限分配 |

### 角色 6：普通企业员工（无招聘业务）

| 维度 | 内容 |
|------|------|
| 侧边栏 | 完全隐藏「招聘管理」一级菜单 |
| 可见内容 | 个人中心：本人投递简历记录 + 本人面试记录 |
| 不可进入 | 任何招聘模块页面 |

## 三、核心业务流程（财务部招出纳示例）

```
1. 财务部普通员工 → 在需求管理提交「出纳」招聘需求
2. 财务部负责人 → 在需求管理审批该需求
3. HR → 查看待审批需求，集中处理
4. 审批通过 → 系统自动匹配内部员工 + HR 同步筛选外部简历
5. HR → 发起面试，指定面试官（财务负责人/业务骨干）
6. 被指定面试官 → 登录后临时出现「面试计划」菜单
7. 面试官 → 查看分配面试 + 填写评价
8. HR → 根据评价结果发起 Offer
9. 流程闭环 → 临时面试官权限自动回收
```

## 四、页面设计规范

### 4.1 页面职责（单一职责拆分）

| 页面 | 职责 | 说明 |
|------|------|------|
| 招聘看板 | 只读数据大盘 | 全局/部门监控 + 风险预警，无业务操作 |
| 需求管理 | 全生命周期操作 | 提需求、审批、岗位发布。页内 3 个 Tab |
| 人才库 | 候选人统一管理 | 仅 HR/管理员可见。页内 2 个 Tab |
| 面试计划 | 面试闭环 | 安排、评价、Offer。页内 3 个 Tab |
| AI 中心 | 一期占位 | 二期迭代替换 |
| 基础配置 | 一期占位 | 二期迭代替换 |

### 4.2 视觉统一规范

| 元素 | 规范 |
|------|------|
| 卡片 | 圆角 12px、内边距 20px、边框左侧 3px 状态色 |
| 状态色 | 待处理 #FF9500 / 进行中 #007FFF / 完成 #00B42A / 拒绝 #F53F3F |
| 按钮 | 主操作蓝色填充 / 成功绿色填充 / 危险红色 / 文字按钮无填充 |
| 表格 | 行高 48px、操作列按钮统一尺寸 |
| 间距 | 模块间 20px、页面四周 24px |
| 信息层级 | 筛选栏 → Tab → 指标卡片 → 表格 → 底部统计 |

### 4.3 数据权限分层

同一页面根据角色自动过滤数据，不展示无权限内容。同一 URL，不同角色看到的数据范围不同。

## 六、关键技术规则

### 6.1 临时面试官权限生命周期

```
HR 发起面试指定面试官
  → 被指定者登录 → 侧边栏自动出现「面试计划」
  → 面试时间到 → 填写评价 → 提交
  → 权限即时回收，菜单消失，恢复原身份
```

### 6.2 部门负责人查看简历的入口

```
需求管理 → 需求列表 → 点击某条需求 → 需求详情
  ├── Tab「外部候选人」→ 本岗位投递/筛选/面试中的外部候选人
  └── Tab「内部可转岗员工」→ 系统匹配的本部门在职员工（JD 自动匹配）
无独立「人才库」菜单入口
```

### 6.3 菜单嵌套统一

所有角色侧边栏结构一致：
```
招聘管理 ▾
  ├── 招聘看板
  ├── 需求管理
  ├── 人才库        ← 无权限则不渲染
  ├── 面试计划      ← 无权限则不渲染
  ├── AI 中心       ← 无权限则不渲染
  └── 基础配置      ← 无权限则不渲染
```
不因权限变化而改变布局结构。
