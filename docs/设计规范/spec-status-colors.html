❓ 为什么是 5 色而非 4 色？
若只保留红/橙/绿/灰四色，橙色将同时承载「待审批(等待)」和「招聘中(执行中)」两种不同语义，造成混淆。新增蓝色后：🟠 橙 = 等待动作（待审批/待安排/待评价），🔵 蓝 = 正在执行（审批中/招聘中/已安排），语义不再混淆。
⚠️ 易混淆点：「待审批」和「审批中」是两个不同的状态——待审批 = 审批节点到达某人，该人需要操作（🟠 橙色）；审批中 = 审批流正在流转中，但当前节点还未到达当前用户（🔵 蓝色）。同理，「待评价」= 面试官需要填评价（🟠 橙色），「已安排」= 面试时间已定但还没到（🔵 蓝色）。

整改前后 对比

❌ 整改前（混乱）

· 招聘中 = 绿色（和"完成"撞色）
· 审批中 = 橙色（和"待办"同色）
· 待评价 = 蓝色（语义不明）
· 已关闭 = 红色（过度警示）
· 各页面自定义颜色，6 种绿色
· 卡片彩色左边框 + 渐变 + 阴影
· 进度条独立大色块，视觉割裂

✅ 整改后（统一）

· 招聘中 = 蓝色（进行中）
· 审批中 = 蓝色（进行中）
· 待评价 = 橙色（需行动）
· 已关闭 = 灰色（非活跃）
· 全系统仅 5 种状态色
· 卡片 8px 圆角 + #eee 细边框
· 单行极简进度指示器
