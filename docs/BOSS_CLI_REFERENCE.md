# Boss CLI 官方使用文档

> 版本: @joohw/boss-cli 0.6.6
> 获取日期: 2026-07-18
> 说明: Boss 直聘浏览器自动化 CLI，纯 CLI 无 Agent 运行时

---

## 基础命令

| 命令 | 说明 |
|------|------|
| `boss` | 进入交互模式，提示符 `boss>`，`exit`/`quit` 退出 |
| `boss help` | 显示帮助 |
| `boss version` / `boss -v` / `boss --version` | 显示版本并检查更新 |
| `boss update` | npm 安装最新版 |

---

## 登录

```
boss login
```
打开登录页，用户需在浏览器中自行完成登录，命令直接返回。

---

## 聊天与候选人管理

### `boss list [--unread]`
读取「全部」聊天列表候选人。`--unread` 仅显示未读（角标 > 0）。

### `boss chat <姓名> [--strict]`
打开指定联系人会话。默认包含模糊匹配，`--strict` 精确匹配。

### `boss chat [姓名] --index <序号> [--unread] [--strict]`
按 `boss list` 输出的 1-based 序号打开会话。
- `--unread`: 序号对应 `boss list --unread`
- 同时提供姓名时校验该序号候选人姓名
- `--strict`: 精确校验
- **仅用于已建立联系的候选人**（在 list 里可见的会话对象）

## 聊天内操作

### `boss action <操作> [--remark <备注>]`
仅在当前聊天页已打开候选人详情时执行操作，只返回 action 执行结果。

| 操作 | 说明 |
|------|------|
| `resume` | 查看简历 |
| `not-fit` | 标记不合适 |
| `remark` | 添加备注（需 `--remark`） |
| `agree-resume` | 同意简历 |
| `request-attachment-resume` | 工具栏「求简历」，确认后发送默认话术索要附件简历（需双方各至少发过一条消息） |
| `history` | 查看历史 |
| `wechat` | 交换微信 |

### `boss send [--text <内容>] [-t <内容>] [--request-resume]`
发送文本消息。
- `--request-resume`: 发送后延迟片刻自动执行「求简历」操作

## 职位管理

### `boss positions`
读取当前职位列表（含开放/待开放/已关闭状态）。

### `boss jd <name>`
抓取指定职位详情并缓存到项目目录同名 `.md` 文件。

## 搜索与推荐

### `boss recommend [岗位关键字]`
进入推荐页并读取推荐列表。带岗位关键字时先在岗位下拉中模糊匹配并切换。

### `boss search [关键词]`
进入「搜索」页并读取 Boss 默认常规搜索结果。带关键词时填入搜索框并回车搜索。

### `boss deep-search [岗位关键字] [--job <岗位关键字>] [--core <核心要求>] [--bonus <加分项>] [--clear-core] [--clear-bonus] [--match]`
进入「深度搜索」页。参数说明：
- `--job`: 岗位关键字，模糊匹配并切换
- `--core`: 核心要求（可重复，按传入列表同步对应分组）
- `--bonus`: 加分项（可重复）
- `--clear-core`: 清空核心要求分组
- `--clear-bonus`: 清空加分项分组
- `--match`: **显式提供时才点击「立即匹配」**，消耗今日匹配次数，输出列表顶部最新 20 条

## 简历预览

### `boss preview <姓名>`
在线简历预览。**须当前已在「推荐」(`/web/chat/recommend`)、「深度搜索」(`/web/chat/aiform`) 或「常规搜索」(`/web/chat/search`) 且列表已加载。**
不会自动跳转。

> ⚠️ 平台对在线简历每日可查看次数有限，请按需使用、谨慎查看。

---

## 打招呼

### `boss greet <姓名> [--job <岗位关键字>]`
须当前已在「推荐」或「深度搜索」且列表已加载。对当前列表中的候选人点击"打招呼"。
- `--job`: 先在岗位下拉中模糊匹配并切换
- **会消耗打招呼次数且单次成本较高，请谨慎使用**

---

## 重要限制

1. **无官方聊天接口**: BOSS 直聘不存在自动 AI 对话 API，HR 必须人工在会话中与候选人沟通
2. **风控机制**: 平台风控持续更新，部分操作可能受限，请先检查版本更新
3. **简历查看次数**: 每日有限，按需使用
4. **打招呼次数**: 有消耗限制，单次成本较高
5. **深度搜索匹配次数**: 每日有限，`--match` 才消耗
